Beginner guide

Test safely before live trading

Start with fake-money and dry-run workflows. Do not add real exchange API keys or run live modes until you understand the logs, dashboard, fees, slippage, and exchange permissions.

Start with paper trading Open Dashboard

Quick mental model

Mode What it does Safe for beginners?
bots.demo_fake_bot Standalone demo with live market data and fake money. No exchange API keys needed. Yes
fake-money Full bot session using FakeMoneyBot. Records sessions/trades for dashboard review. Yes
--dry-run Safety flag that forces classic or delta-neutral commands to use FakeMoneyBot instead of real orders. Yes
classic / delta-neutral Live trading modes. These can place real orders when API keys and permissions are configured. No
Recommended path: demo first, then fake-money, then dry-run using the same command shape as live trading, then review the dashboard before considering real orders.

1. Install

git clone https://github.com/nguyenngocbinhneu/bmlb-arbitrage-bot.git
cd bmlb-arbitrage-bot
pip install -r requirements.txt

For the first run, do not create a .env file with real trading keys. The safe demo does not need it.

Exchange account links

If you do not already have exchange accounts, these referral links can be used to sign up.

Binance Bybit OKX KuCoin
These may be referral links. Check each exchange's fees, regional availability, KYC rules, API permissions, and risk terms before using live trading features.

2. Run the safest demo

This command uses live exchange market data with fake money and does not place real orders.

python -m bots.demo_fake_bot --symbol BTC/USDT --exchanges binance okx bybit --duration 5

Use this to verify that dependencies, exchange connectivity, and your local environment work.

3. Run a full paper trading session

fake-money runs the normal bot flow, but simulated trades are handled by FakeMoneyBot.

python main.py fake-money 15 1000 binance kucoin okx BTC/USDT

After the run, start the web dashboard to inspect sessions, trade records, PnL, fees, and slippage.

python -m uvicorn web.app:app --reload --port 8000

Open http://localhost:8000/dashboard.

4. Use dry-run before any live mode

The --dry-run flag is a guardrail. In this codebase, if --dry-run is present, main.py creates FakeMoneyBot even when the selected mode is classic or delta-neutral. 

# Same shape as classic mode, but no real orders are sent.
python main.py classic 15 1000 binance kucoin okx BTC/USDT --dry-run

# Same shape as delta-neutral mode, but still forced to fake-money behavior.
python main.py delta-neutral 15 1000 binance kucoin okx BTC/USDT --dry-run
Practical use: build and test the exact command you plan to use later, keep --dry-run on, then review dashboard results before removing it.

5. Multi-pair paper trading

Start with small fake balances and a few liquid pairs.

python main.py fake-money 15 1000 binance kucoin okx --symbols BTC/USDT ETH/USDT SOL/USDT

You can also add --dry-run to classic or delta-neutral multi-pair commands to keep them simulated.

Checklist before live trading

  1. You have run fake-money sessions for the same exchanges and symbols.
  2. You have reviewed dashboard PnL, fees, failed trades, and slippage.
  3. You understand exchange API permissions and have disabled withdrawals on API keys.
  4. You have set risk limits in configs.py.
  5. You have tested with the smallest practical trade size.
  6. You accept that live results can differ from paper trading because of latency, liquidity, fees, and partial fills.
Do not skip this: live crypto trading can lose money. This software does not guarantee profit and is not financial advice.
